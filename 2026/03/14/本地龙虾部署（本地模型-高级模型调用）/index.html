本地龙虾部署（本地模型+高级模型调用）

最近龙虾很火，很多人都想尝试，但是又担心主力电脑的个人信息泄露，出现巨大的安全漏洞。

除了使用各大厂商提供的整套的云服务方案之外（贵啊！！！！）

我给你一套最简单、用完就关、零浪费、绝对安全的低成本

「高配机做服务端 + 闲置机内网调用」一键方案。

简单任务用 Ollama本地模型，高级任务用各大厂****API。

闲置机 = 只跑「小龙虾」UI / 前端

高配机 = 只跑模型（Ollama 后端）

一、本地需求

一台高级电脑做服务器（平时自用的主力电脑），另一台闲置电脑内网调用（物理隔离）。

高配电脑

入门（千问7B 模型）：RTX 4060 8G + 16G 内存 + 512G SSD → 约 6,000–8,000 元（本案例以7B模型为例）

主流（千问7B/14B）：RTX 4060 Ti 16G + 32G 内存 + 1TB SSD → 约 10,000–12,000 元

高端（千问32B）：RTX 4090 24G + 64G 内存 + 2TB SSD → 约 18,000–22,000 元

闲置电脑

没有重要个人信息的win或者mac都行。没有的话淘个二手的低配笔记本也行，几乎不需要几个钱。

二、整体流程（超短）

  1. 高配机：装 Ollama → 跑模型 → 开内网访问

  2. 闲置机：不用装模型，不用显卡，直接访问

  3. 用完：关掉 Ollama 就行，不耗电

六、多模型配置 + 任务路由 / 手动切换

「按任务难度分流调用不同模型」是非常实用的进阶用法 ——高级任务（复杂推理 / 创作）走 GPT/Gemini API，基础任务（简单问答 / 整理）走本地模型，既省 API 费用又保证效率。

核心实现思路

小龙虾本身支持多模型配置 + 任务路由 / 手动切换，我们只需要做两步：

  1. 在小龙虾中配置两套模型（GPT/Gemini + 本地 Ollama）的 API 信息；

  2. 要么手动选择模型执行任务，要么通过「提示词 / 任务类型」自动分流（进阶）。

一、在小龙虾中配置多模型 API Key

1.基础配置（手动切换模型）

这是最简单的方式，适合新手，步骤如下：

（1）配置 GPT/Gemini API（高级任务）

  • 打开小龙虾（OpenClaw）→ 进入「设置」→「模型配置」；

  • 新增模型：

    • 模型类型：选择「OpenAI（GPT）」、「Google Gemini」或者其他大厂的模型；

    • API 地址：

      • GPT：默认 https://api.openai.com/v1（国内需配置代理）；

      • Gemini：https://generativelanguage.googleapis.com/v1；

    • API Key：填入你申请的 GPT、Gemini 或者其他大厂的模型官方 API Key；

    • 模型名称：例如「GPT-4o」「Gemini 1.5 Pro」，保存。

（2）配置本地 Ollama 模型（低级任务）

  • 继续在「模型配置」中新增模型：

    • 模型类型：选择「OpenAI 兼容」（Ollama 支持 OpenAI 格式）；

    • API 地址：填入高配机的内网地址 http://192.168.1.100:11434/v1；

    • API Key：随便填（如 dummy，Ollama 不校验）；

    • 模型名称：填入你本地的模型名（如「qwen:7b」），保存。

（3）使用时切换

执行任务前，在小龙虾的「任务设置」或「输入框旁」选择对应的模型：

  • 写复杂方案、深度推理 → 选「GPT-4o/Gemini」；

  • 简单问答、文本整理、本地文件处理 → 选「qwen:7b（本地）」。

  1. 进阶配置（自动分流，不用手动切换）

如果想让小龙虾「自动判断任务难度」分流，需要用到小龙虾的「提示词路由」或「插件规则」（以主流版本为例）：

（1）新建规则模板

  • 进入小龙虾「插件 / 自动化」→「新建路由规则」；

  • 设置触发条件：

    • 条件 1：提示词包含「复杂」「推理」「创作」「分析」等关键词 → 路由到「GPT/Gemini」；

    • 条件 2：提示词包含「简单」「整理」「总结」「本地文件」等关键词 → 路由到「本地 Ollama」；

    • 默认规则：无关键词时，优先走本地模型（省 API 费）。

二、关键注意事项（避坑）

  1. 网络适配：

    1. GPT/Gemini API 需要外网 / 代理，本地 Ollama 只需要内网，小龙虾会自动识别不同地址的网络；

    2. 建议给小龙虾配置「双网络」：本地内网（连高配机）+ 代理（连外网 API）。

  2. 费用控制：

    1. 在小龙虾中开启「API 用量统计」，设置 GPT/Gemini 的「月额度上限」（如 50 元），超额度自动切本地模型；

    2. 本地模型无费用，可放心用。

  3. 模型兼容性：

    1. 确保小龙虾选择的「模型名称」与实际一致（如 GPT-4o 不能写成 gpt4，本地 qwen:7b 不能漏后缀）；

    2. Ollama 可通过 ollama list 命令查看本地模型全称。

总结

  1. 基础用法：在小龙虾中分别配置 GPT/Gemini 和本地 Ollama 的 API 信息，手动选择模型执行不同难度任务；

  2. 进阶用法：通过「关键词路由规则」让小龙虾自动判断任务难度，高级活走云端 API，低级活走本地模型；

  3. 核心优势：既利用云端大模型的能力，又通过本地模型节省 API 费用，完全适配你的「低频使用、物理隔离」需求。

六、总结

闲置机装小龙虾 → 指向高配机的 Ollama 地址

高配机只跑模型 → 用完就关机

这就是你要的：物理隔离 + 最低成本 + 最舒服的本地部署。
