2026年我的第一篇技术笔记

2026年我的第一篇技术笔记

2026年开篇，先给自己立一个小目标：把技术学习的过程沉淀下来，于是决定搭建一个属于自己的独立博客。折腾了大半天，从零基础搞定Hexo+GitHub Pages部署，踩了不少坑，也总结了一套能直接落地的流程，特此记录，希望能帮到和我一样的新手，少走弯路。

核心结论先放前面：Hexo+GitHub Pages是新手搭建独立博客的最优解——零服务器成本、操作简单、部署后稳定，全程不用复杂代码，跟着步骤走就能100%成功。

一、前置准备（必做，10分钟搞定）

搭建前需要准备3个工具，全部免费，安装过程一路默认即可，不用额外配置。

  1. Git：用于将本地博客文件推送到GitHub，是部署的核心工具。
    下载地址：https://git-scm.com/download/win（Windows版本），安装时记得勾选「Git Bash Here」，方便后续右键打开命令窗口。
    验证方法：右键桌面能看到「Git Bash Here」，即为安装成功。

  2. Node.js：Hexo是基于Node.js运行的，必须安装对应环境。
    下载地址：https://nodejs.org/（选择LTS版本，比如20.x，稳定不报错），安装时勾选「Add to PATH」，让系统能识别Node命令。
    验证方法：打开Git Bash，输入「node -v」，能显示版本号即为成功。

二、步骤1：创建GitHub Pages仓库（关键，不能错）

GitHub Pages的仓库名有严格要求，错一个字符都会导致博客无法访问，这是我踩的第一个小坑，特此强调。

  1. 登录GitHub，点击右上角「+」→「New repository」，进入仓库创建页面。

  2. 仓库名必须填写：你的GitHub用户名.github.io（比如我的是Snowkey-zju.github.io），这是博客的默认访问地址，不能修改格式。

  3. 勾选「Public」（必须公开，免费账户只有公开仓库才能启用GitHub Pages），其他选项默认，点击「Create repository」，仓库就创建完成了。

三、步骤2：本地搭建Hexo博客（纯复制命令，零技术门槛）

这一步是在本地创建博客的雏形，所有操作都在Git Bash中完成，命令我已经整理好，直接复制粘贴即可，不用自己手写。

  1. 在桌面右键→「Git Bash Here」，打开命令窗口，执行以下命令，创建并进入博客文件夹（文件夹名可自定义，我用的是myblog，注意不要有中文和空格）：mkdir myblog && cd myblog

  2. 安装Hexo并初始化博客，依次执行以下3条命令，耐心等待安装完成（中间出现警告不用管，不影响使用）： npm install -g hexo-cli hexo init npm install

  3. 本地预览博客，验证是否搭建成功： hexo server 执行完成后，打开浏览器访问「http://localhost:4000」，能看到Hexo的默认页面，说明本地博客搭建成功了！按「Ctrl+C」可以停止本地预览。

四、步骤3：配置Hexo部署到GitHub（核心步骤，避坑重点）

这一步是将本地的博客，发布到GitHub Pages，让别人能通过域名访问。这里容易出现格式错误和插件安装失败，我整理了最稳的操作。

  1. 安装部署插件（Hexo本身没有Git部署功能，必须单独安装）： npm install hexo-deployer-git --save 看到「added xx packages」提示，说明插件安装成功。如果安装失败，可先执行「npm uninstall hexo-deployer-git」卸载残留，再重新安装。

  2. 修改Hexo配置文件（_config.yml），这是最容易出错的一步，重点注意格式！

    1. 打开本地博客文件夹（myblog），找到「_config.yml」文件，用记事本或VS Code打开。

    2. 滑到文件最底部，找到「deploy」部分，修改为以下内容（替换成你的GitHub仓库地址）： deploy: type: git repo: https://github.com/Snowkey-zju/Snowkey-zju.github.io.git # 替换成你的仓库地址 branch: main

    3. ⚠️ 关键提醒：yaml文件的格式要求严格，冒号后面必须加一个空格（比如「type: git」，不能写成「type:git」），否则会报错「Deployer not found: git」。

五、步骤4：发布博客到GitHub Pages（一键上线）

配置完成后，执行一条命令就能将本地博客发布到互联网，首次部署会弹出GitHub登录窗口，登录你的账号即可。

  1. 在Git Bash中执行发布命令（清理缓存+生成静态文件+部署）： hexo clean && hexo generate && hexo deploy

  2. 等待部署完成，看到「Deploy done: git」提示，说明发布成功。

  3. 验证是否上线：等待1-3分钟（GitHub Pages生效需要时间），打开浏览器访问「https://你的用户名.github.io」（比如我的是https://Snowkey-zju.github.io），能看到和本地预览一样的页面，就说明博客正式上线了！

六、新手必踩的5个坑（我亲身踩过，帮你避坑）

  1. 仓库名错误：必须是「用户名.github.io」，多一个字、少一个字，或者大小写错误，都会导致404页面。

  2. 配置文件格式错误：_config.yml中冒号后没加空格，会导致部署失败，报错「Deployer not found: git」。

  3. 博客文件夹有中文/空格：Windows系统下，文件夹路径含中文或空格（比如「我的博客」「my blog」），会报错「Spawn failed」，建议改成纯英文（比如myblog）。

  4. 仓库设为私有：免费账户的GitHub Pages仓库必须设为Public，设为Private会导致博客无法访问，报错「There isn’t a GitHub Pages site here」。

  5. 部署后404：除了上述原因，还可能是GitHub Pages未开启，进入仓库→Settings→Pages，确认Source选择「main」分支和「/(root)」文件夹，保存后等待几分钟即可。

七、日常写作与发布流程（简化版）

博客搭建完成后，日常写作就很简单了，我整理了3步核心流程，不用记复杂命令：

  1. 新建文章：在Git Bash中执行「hexo new “文章标题”」，会在「source/_posts」文件夹生成.md文件，用Markdown编辑器写内容即可。

  2. 本地预览：执行「hexo server」，访问「http://localhost:4000」，确认排版和内容无误。

  3. 发布上线：执行「hexo clean && hexo generate && hexo deploy」，等待1-3分钟，博客就更新完成了。

进阶技巧：可以创建一个批处理文件（deploy.bat），把发布命令写进去，以后双击就能一键发布，不用每次复制命令（具体脚本我会在后续笔记中分享）。

八、总结

2026年的第一篇技术笔记，以搭建独立博客收尾，整个过程不算复杂，但细节很重要——很多报错都是因为格式错误、路径问题导致的，只要跟着步骤走，避开我踩过的坑，新手也能1小时内完成部署。

独立博客的意义，在于拥有一个完全属于自己的内容阵地，不用受平台规则限制，把学习笔记、技术心得沉淀下来，既是对自己的总结，也能帮助他人。后续我会继续分享博客美化、图片管理、多电脑同步等技巧，慢慢把这个小阵地打磨得更好。

最后，附上我的博客地址：https://Snowkey-zju.github.io，欢迎访问交流～
